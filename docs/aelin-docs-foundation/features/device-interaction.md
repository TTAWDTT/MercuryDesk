# Device Interaction

## 已有能力（按平台可用性）

- 读取进程列表（CPU/内存排序）
- 异常占用识别
- 进程动作（结束/调低优先级/调高优先级）
- 模式切换（meeting / focus / sleep / normal）

## 典型场景

- 卡顿时快速识别异常进程
- 专注工作时降低干扰
- 夜间使用时降低亮度并减少通知

## 注意

设备控制能力受操作系统和权限影响，某些动作会回退为“状态记录但不执行系统级变更”。
