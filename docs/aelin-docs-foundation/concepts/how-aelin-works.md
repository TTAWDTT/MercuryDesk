# How Aelin Works

Aelin 的标准链路是：

1. `理解问题`：识别意图、时效性、是否需要外部证据。
2. `任务规划`：拆出本地查询与网络查询边界（可并行）。
3. `回答生成`：先给结论，再附证据卡片与可执行动作。
4. `记忆更新`：将高价值结果沉淀为可复用长期记忆。

## 工具调用原则

- 不是每次都上网。
- 不是每次都跟踪。
- 由模型基于问题语义和证据覆盖度动态决策。

## 跟踪触发原则

满足以下任意特征时，会倾向建议跟踪：

- 明确对象（账号、球队、作者、关键词）
- 明确时间连续性（最近、持续、后续、每天）
- 用户反复追问同一主题

## 主动交互原则

当出现高价值更新、状态变化或异常事件时，Aelin 可主动提醒，而不是等你再次提问。
