## 一、信息摘要
- 调用LLM生成信息摘要
- 输入：该信息/帖子/视频/图片的内容 + 上下文（用户偏好）
- 输出：摘要
- 难点：视频总结
- 可选：只对**长文本/视频**做总结，其余内容不呈现总结
## 二、草拟回复
- 调用LLM生成回复（仅针对邮件信息）
- 输入：同（1）
- 输出：草拟内容。并允许用户在该页面发送
- 难点：允许LLM自行地读取邮件内部带有的附件
## 三、信息助手1-总assistant
- 调用LLM，可以进行chat
- 输入：用户的human input + system prompt1 + 记忆1：用户偏好与近况（集成sama）+ 记忆2：所有message内容 + 记忆3：短期聊天记忆
- 输出：对于问题的解答
## 四、信息助手2-专一信息助手
- 同三，但是只针对单一message
- 若实现该功能，可以去除功能1、2，且效果会大于1+2
- 输入：用户的human input + system prompt2 + 记忆1：用户偏好与近况（集成sama）+ 记忆2：所有message内容（注：此处依旧注入所有message） + 记忆3：短期聊天记忆
- 输出：对于该邮件的回答
## 五、Message自动排序与置顶
- 对message进行排序和置顶
- 输入：当前所有message + message更新时间/是否已读 + 用户偏好
- 输出：自动排序与置顶
- 【注】到这里开始要关注一个问题，关于用户偏好，应当由两部分组成，一部分是agent的记忆，可以反映出用户的关注度，另一部分是用户的点击频率/停留时长等等，此为常规做法。
## 六、每轮刷新后的‘报纸’
- 每轮刷新后会为该次刷新做出一个总结报纸悬浮窗呈现在前端
- 输入：略
- 输出：报纸
